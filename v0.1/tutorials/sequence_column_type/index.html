The bioseq.seq column type#

The Tbioseq.seq type is parameterised by an alphabet and a seed width. Its anatomy:

bioseq.seq('dna4', 64)
alphabet

Which letters are legal and how densely they pack. 'dna4' packs 2 bits per base.

seed width (k)

The k-mer width the index seeds with. Larger k is more specific; smaller k is more sensitive.

i

Note. The alphabet and seed width are fixed at column-creation time. Sequences you insert are validated and packed against them on the way in.

Streaming FASTA & FASTQ#

ƒbioseq.load() reads records incrementally, so memory stays flat regardless of file size. Read the call left-to-right:

load_mapped.sql 
SELECT bioseq.load(
  'reads'::regclass,           -- target table
  '/data/run_042.fastq',     -- source path
  'auto',                    -- detect fasta/fastq
  ARRAY[
    ('sample_id', 'sample')::bioseq.column_mapping,
    ('run_name',  'run')::bioseq.column_mapping
  ]
);
regclass

Pass the table as a regclass so the loader resolves its columns and the sequence type.

column_mapping[]

Each tuple binds a table column to a header field, so metadata arrives typed alongside each sequence.

Searching with mmsearch#

Once loaded, project the alignment accessors right beside the row. Each accessor takes the row's primary key and returns a facet of the match found by ƒmmsearch().

alignment.sql 
SELECT pk,
       bioseq.query_start(pk)  AS qs,
       bioseq.query_end(pk)    AS qe,
       bioseq.target_start(pk) AS ts,
       bioseq.target_end(pk)   AS te,
       bioseq.cigar(pk)        AS cg
FROM   reads
WHERE  mmsearch(seq0, 'ACGTACGTACGT'::bioseq.seq('dna4', 64))
ORDER BY pk;
