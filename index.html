Reading the query

One SELECT, now sequence-aware

A normal query becomes a sequence search the moment mmsearch() appears in the WHERE clause. Everything else is SQL you already write.

query.sql 
SELECT id, accession
FROM landing_hero_sequences
WHERE mmsearch(seq, 'CTAGTCACTGTAGCCTGATCGTAGCTAGT'::bioseq.seq('dna4', 64))
ORDER BY id;
Executed query result
id accession
1042 NR_046018.2
  • SELECT

    Select any metadata from the table just like ordinary Postgres. Join, filter, and project columns exactly as you normally would.

  • WHERE mmsearch()

    Adding mmsearch() to the WHERE clause activates a custom scan that performs the alignments efficiently.

  • bioseq.seq('dna4', 64)

    Bioseq supports different sequence alphabets and storage widths, chosen right in the column type modifier.

One workflow

Load, query, and scale in one place

Every step is ordinary SQL. Nothing leaves the database, so sequence work and metadata work stay together.

  1. 01 Load Stream FASTA and FASTQ files straight into tables.
  2. 02 Query Find sequence matches with mmsearch inside a normal SELECT.
  3. 03 Filter Combine matches with metadata filters in the same query.
  4. 04 Inspect Read alignment context such as match start and end.
  5. 05 Scale Work with very large records without splitting them by hand.

How the index works

From long sequences to small, searchable seeds

Before search runs, BioseqDB turns each sequence into compact seeds. Minimizers, spaced seeds, and strobemers reduce long records to stable lookup keys, which is why search stays fast as collections grow.

  1. 01

    Minimizers

    Sliding windows keep the lowest-hash k-mer, reducing a long sequence to a compact, stable set of representatives.

    ACGTACG
  2. 02

    Spaced seeds

    A mask samples selected positions inside each seed, tolerating substitutions without giving up fast indexed lookup.

    A·G·A·G
  3. 03

    Strobemers

    Distant k-mers are linked into one seed, staying distinctive even when an insertion or deletion shifts the sequence between them.

    ACGTGA

Built for big records

One record, any size

Bioseq stores sequence lengths across the full 64-bit range. Very large sequences are sharded internally, so a long chromosome or assembly stays a single row.

  • 264 Sequence length range, the full 64-bit class
  • 1 row Per record, however long
  • auto Internal sharding, no manual splits
