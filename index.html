SQL-native sequence search

Search biological sequences with plain SQL.

BioseqDB is a SQL-native system for biological sequence data. Load FASTA and FASTQ files, store sequences beside their metadata, and search them with one query.

Reading the query

One SELECT, now sequence-aware

A normal query becomes a sequence search the moment mmsearch() appears in the WHERE clause. Everything else is SQL you already write.

query.sql 
SELECT id, accession
FROM landing_hero_sequences
WHERE public.mmsearch(seq, 'CTAGTCACTGTAGCCTGATCGTAGCTAGT'::bioseq.seq('dna4', 64))
ORDER BY id;

The SQL

Three queries, one product story

Start with the smallest search, add metadata filters, then load a file and keep indexes current. Same language throughout.

query.sql 
SELECT id, accession
FROM landing_hero_sequences
WHERE public.mmsearch(seq, 'CTAGTCACTGTAGCCTGATCGTAGCTAGT'::bioseq.seq('dna4', 64))
ORDER BY id;
CREATE TABLE reference_reads (
  id                bigint PRIMARY KEY,
  accession         text,
  organism          text,
  collected_on      date,
  quality_score     double precision,
  seq               bioseq.seq('dna4', 64)
) USING bioseq;

INSERT INTO reference_reads VALUES
  (5521, 'NR_046018.2', 'Homo sapiens', DATE '2026-06-10', 99.5,
   (repeat('A', 600) || 'CTAGTCACTGTAGCCTGATCGTAGCTAGT' || repeat('C', 600))::bioseq.seq('dna4', 64)),
  (4410, 'NC_000913.3', 'Homo sapiens', DATE '2025-11-04', 97.0,
   (repeat('C', 520) || 'CTAGTCACTGTAGCCTGATCGTAGCTAGT' || repeat('G', 540))::bioseq.seq('dna4', 64)),
  (6193, 'CP041828.1', 'Homo sapiens', DATE '2025-02-18', 94.5,
   (repeat('G', 500) || 'CTAGTCACTGTAGCCTGATCGTAGCTAGT' || repeat('T', 700))::bioseq.seq('dna4', 64)),
  (7102, 'FILTERED.1', 'Mus musculus', DATE '2026-07-01', 100.0,
   (repeat('T', 600) || 'CTAGTCACTGTAGCCTGATCGTAGCTAGT' || repeat('A', 600))::bioseq.seq('dna4', 64));

SELECT id,
       accession,
       organism,
       bioseq.len(seq) AS bases
FROM reference_reads
WHERE organism = 'Homo sapiens'
  AND collected_on >= DATE '2025-01-01'
  AND bioseq.len(seq) >= 1000
  AND public.mmsearch(
        seq,
        'CTAGTCACTGTAGCCTGATCGTAGCTAGT'::bioseq.seq('dna4', 64)
      )
ORDER BY quality_score DESC, accession
LIMIT 25;
CREATE TABLE run_reads (
  row_no        bigserial PRIMARY KEY,
  read_id       text NOT NULL,
  sample_id     text NOT NULL,
  run_name      name NOT NULL DEFAULT 'run_042',
  collected_on  date NOT NULL DEFAULT CURRENT_DATE,
  seq           bioseq.seq('dna4', 64)
) USING bioseq;

SET bioseq.update_index_on_insert = on;

SELECT bioseq.load(
  'run_reads'::regclass,
  'landing_reads.fasta',
  'uni_prot',
  ARRAY[
    ('read_id', 'accession')::bioseq.column_mapping,
    ('sample_id', 'organism_name')::bioseq.column_mapping
  ]
);

INSERT INTO run_reads (read_id, sample_id, run_name, seq)
VALUES (
  'manual_read_001',
  'validation_batch',
  'run_042',
  'CTAGTCACTGTAGCCTGATCGTAGCTAGT'::bioseq.seq('dna4', 64)
);

SELECT read_id,
       sample_id,
       bioseq.target_start(row_no) AS match_start,
       bioseq.target_end(row_no)   AS match_end
FROM run_reads
WHERE public.mmsearch(
        seq,
        'CTAGTCACTGTAGCCTGATCGTAGCTAGT'::bioseq.seq('dna4', 64)
      )
ORDER BY collected_on DESC, read_id;

How the index works

From long sequences to small, searchable seeds

Before search runs, BioseqDB turns each sequence into a set of compact seeds. It picks representative k-mers as minimizers, tolerates gaps with spaced seeds, and links distant k-mers into strobemers. Those seeds are what mmsearch looks up, which is why search stays fast as collections grow.

01

Minimizers

A minimizer is the most compact way to seed a sequence. BioseqDB slides a short window along the read, hashes every k-mer inside it, and keeps only the one with the smallest hash. Neighboring windows usually agree on that k-mer, so a long sequence reduces to a small, stable set of seeds.

Spaced seeds

A spaced seed keeps some positions and ignores others along a fixed pattern. Two k-mers that differ only at the ignored positions map to the same seed, so a single mismatch does not hide a real match.

Strobemers

A strobemer links several short k-mers taken from gapped positions into one seed. Spreading the pieces out covers a wider span while staying small, which helps matches survive insertions and deletions.

Built for big records

One record, any size

BioseqDB stores sequence lengths in a 64-bit range, up to the 264 class. Very large sequences are sharded internally, so a long chromosome or assembly stays a single row.

You never split a record by hand to make it fit. The storage model does that work for you.
